# Черновик для руководства пользователя

## Наличие контроля: `да`, `пусто`, `отменен`

### Основные состояния

| Состояние | Смысл | Можно создавать заявку/результат | Как участвует в отчетах |
| --- | --- | --- | --- |
| `да` | Контроль или ПСТО требуется | Да | Стык участвует в ЛНК/ПСТО как активный |
| `пусто` | Контроль или ПСТО не требуется | Нет | Стык не должен участвовать в соответствующем отчете |
| `отменен` | Контроль или ПСТО отменили, но историю нужно сохранить/показать | Нет | Стык остается видимым в отчете с отметкой отмены |

### Правила по кейсам

| Сценарий | Что выбираем в наличии | Можно сохранить | Что делает система | Что увидит пользователь |
| --- | --- | --- | --- | --- |
| По виду контроля/ПСТО ничего не заполнено | `да` | Да | Контроль становится активным | Стык можно добавить в заявку и результат |
| По виду контроля/ПСТО ничего не заполнено | `пусто` | Да | Контроль считается ненужным | Стык не участвует в соответствующем отчете |
| По виду контроля/ПСТО ничего не заполнено | `отменен` | Да | Контроль считается отмененным | Стык отображается с отметкой `отменен`, но новую заявку/результат создать нельзя |
| ЛНК: есть только заявка, результата/даты/заключения нет | `пусто` | Да | Заявка ЛНК по этому виду контроля будет удалена автоматически | Подсказка: `ВИК: заявка на стык будет удалена` |
| ПСТО: есть только заявка и/или дата, результата/диаграммы нет | `пусто` | Да | Заявка ПСТО и дата ПСТО будут очищены автоматически | Подсказка: `ПСТО: заявка и дата на стык будут удалены` или `ПСТО: заявка на стык будет удалена` |
| ЛНК: есть результат контроля, дата контроля или заключение | `пусто` | Нет | Сохранение блокируется, чтобы не потерять выполненный контроль | Подсказка: `ВИК: выберите «отменен» либо очистите/удалите результат НК в отчете ЛНК.` |
| ПСТО: есть результат `проведено` или диаграмма | `пусто` | Нет | Сохранение блокируется, чтобы не потерять выполненную операцию | Подсказка: `ПСТО: выберите «отменен» либо очистите/удалите результат ПСТО.` |
| ЛНК: есть только заявка, результата/даты/заключения нет | `отменен` | Да | Заявка ЛНК будет удалена автоматически, новых заявок/результатов по этому виду создать нельзя | В отчете вид контроля будет отображаться как `отменен` |
| ПСТО: есть только заявка и/или дата, результата/диаграммы нет | `отменен` | Да | Заявка ПСТО и дата ПСТО будут очищены автоматически, новый результат ПСТО создать нельзя | В отчете ПСТО будет отображаться `отменен` |
| ЛНК: есть положительный результат `годен` | `отменен` | Да | Заявка, дата контроля и заключение сохраняются | Результат отображается как `годен (отменен)` |
| ПСТО: есть положительный результат `проведено` | `отменен` | Да | Заявка, дата ПСТО и диаграмма сохраняются | Результат отображается как `проведено (отменен)` |
| ЛНК: есть отрицательный результат `ремонт` или `вырез` | `отменен` | Да | Заявка, дата контроля и заключение очищаются; отрицательный факт результата остается в данных для логики цепочек | В отчете такой вид отображается как `отменен`, итоговый статус больше не ломается из-за отмененного контроля |
| Наличие `пусто`, но результат заполнен | Не пользовательский сценарий | Нет/ошибка данных | Система считает данные противоречивыми | Итоговый статус становится `ошибка`, при наведении показывается причина |

### Почему может появиться `пусто`, но результат заполнен

| Источник | Пример |
| --- | --- |
| Старые тестовые данные | Данные были созданы до появления текущих правил |
| Импорт | В Excel пришел результат, но не заполнено наличие контроля |
| Ручное изменение в БД | Данные изменили напрямую, минуя интерфейс |
| Старый сценарий работы | Ранее логика могла автоматически чистить или менять поля иначе |

### Подсказки в меню редактирования стыка

| Ситуация | Тип подсказки | Текст подсказки |
| --- | --- | --- |
| Пользователь ставит `пусто`, а по виду НК есть только заявка | Предупреждение перед сохранением | `При сохранении: ВИК/РК/...: заявка на стык будет удалена` |
| Пользователь ставит `пусто`, но по виду НК уже есть результат, дата контроля или заключение | Блокировка сохранения | `ВИК/РК/...: выберите «отменен» либо очистите/удалите результат НК в отчете ЛНК.` |
| Пользователь ставит `пусто`, а по ПСТО есть заявка и/или дата без результата | Предупреждение перед сохранением | `При сохранении: ПСТО: заявка и дата на стык будут удалены` |
| Пользователь ставит `пусто`, но по ПСТО уже есть результат или диаграмма | Блокировка сохранения | `ПСТО: выберите «отменен» либо очистите/удалите результат ПСТО.` |
| Пользователь ставит `отменен`, а по ЛНК есть положительный результат | Информационная логика отчета | В отчете будет показано `годен (отменен)` |
| Пользователь ставит `отменен`, а по ПСТО есть положительный результат | Информационная логика отчета | В отчете будет показано `проведено (отменен)` |
| Итоговый статус `ошибка` | Подсказка при наведении | Показывается, где результат заполнен, но наличие стоит `пусто` |

### Короткая логика для пользователя

| Нужно сделать | Правильное действие |
| --- | --- |
| Контроль больше не нужен, и по нему есть только заявка | Поставить `пусто`; заявка удалится автоматически |
| Контроль уже выполнен, но потом был отменен заказчиком/по решению проекта | Поставить `отменен`; выполненный положительный результат останется для предъявления |
| Контроль уже выполнен, но нужно полностью удалить его из истории | Сначала очистить/удалить результат в отчете ЛНК или ПСТО, затем менять наличие |
| В отчете появился статус `ошибка` | Навести курсор на `ошибка`, посмотреть причину и исправить наличие или результат |
